# SAIVerse 2月リリース計画 (2026年)

作成日: 2026-01-29
目標: 2月中に他ユーザーへの配布可能な状態にする

## 概要

SAIVerseを一般ユーザーに配布するための準備を行う。自律稼働など高度な機能は実験的機能として段階的にリリースし、まずは基本的な対話体験とカスタマイズ性を確保する。

---

## フェーズ1: 基盤機能の整備 (1月末〜2月第2週)

### 1.1 Playbook手動選択機能 【優先度: 最高】

**現状の課題**
- Qwenによる自動選択に依存しており、狙ったPlaybookが実行されないことがある
- 軽量とはいえ自動選択はAPI利用を必要とする
- 配布版ではユーザーが明示的に指定できるモードが必要

**実装内容**
- [ ] UIからPlaybookを直接指定して実行する機能
  - コマンド形式（例: `/playbook search_web`）
  - ボタン/ドロップダウンからの選択
- [ ] Scheduleへの適用（ルーターをバイパスしてサブPlaybookを直接指定）
- [ ] 自動選択モードとの切り替え

**完了条件**
- ユーザーが任意のPlaybookを手動で実行できる
- Scheduleで指定したPlaybookが確実に実行される

### 1.2 Schedule連携 【優先度: 高】

**現状の課題**
- メール送信スケジュールでルーターがbasic_chatを選択してしまうことがある

**実装内容**
- [ ] Playbook手動選択機能をScheduleに統合
- [ ] スケジュール設定時にPlaybookを指定可能に

**完了条件**
- 毎朝のニュース検索→メール送信が確実に動作する

## フェーズ2: Item機能の整理 (2月第2週)

### 2.1 Item操作Playbookの統合 【優先度: 中】

**現状の課題**
- Item操作系のPlaybookが乱立している（view/pickup/place/use等）
- ユーザーにとって分かりにくい

**実装内容**
- [ ] 操作系Playbookを統合または整理
- [ ] ItemのOpen状態をペルソナから自然に制御できるようにする
- [ ] コンテナItem（本棚・物置）の実装
  - Itemを格納できるItem
  - ディープリサーチ結果の整理に使用

**完了条件**
- Item操作が直感的に行える
- 部屋が散らからない整理手段がある

## リスクと対策

| リスク | 対策 |
|--------|------|
| Playbook手動選択の実装が想定より複雑 | 最小限のCLI実装から始める |
| オンボーディングドキュメントの量が多い | 必須部分を優先、詳細は後から追加 |
| 自律稼働のAPIコストが高い | 実験的機能として明記、警告を表示 |
| ユーザーからのフィードバック対応 | リリース後1週間はサポート優先期間 |
